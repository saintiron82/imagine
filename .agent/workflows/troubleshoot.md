# 트러블슈팅 워크플로우

테스트 실패 또는 개발 중 문제 발생 시 이 워크플로우를 실행합니다.

## 실행 단계

### 1. 문제 기록
1. `docs/troubleshooting.md` 파일을 엽니다.
2. 다음 템플릿으로 문제를 기록합니다:

```markdown
## [YYYY-MM-DD] U-XXX: 문제 제목

### 증상
- 에러 메시지 또는 예상과 다른 동작

### 원인 분석
- 근본 원인 파악

### 해결 방법
- 적용할 해결책

### 예방책
- 재발 방지를 위한 조치
```

### 2. 원인 분석
3. 에러 로그 확인
4. 관련 코드 검토
5. 의존성/환경 문제 확인

### 3. 해결 시도
6. 코드 수정
7. 재테스트 실행

### 4. 결과 확인
- ✅ 해결됨 → troubleshooting.md에 해결 방법 기록 → 원래 유닛의 테스트 단계로 복귀
- ❌ 미해결 → 재분석 또는 사용자에게 방향 재검토 요청

## 주의사항
- 3회 연속 실패 시 반드시 사용자에게 알림
- 해결 없이 다음 유닛으로 진행 금지
