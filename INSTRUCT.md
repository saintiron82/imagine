# ImageParser 개발 지침서 (INSTRUCT)

> [!IMPORTANT]
> **이 문서는 모든 개발 작업의 시작점입니다.**
> 개발 에이전트는 반드시 이 지침을 따라야 합니다.

---

## 🤖 에이전트 행동 규칙

### 필수 워크플로우 사용
개발 작업 시작 전, **반드시** 아래 슬래시 커맨드를 사용합니다:

```
/unit-start    → 유닛 개발 시작 (5단계 프로세스)
/unit-status   → 현재 진행 상태 확인
/unit-done     → 유닛 완료 및 회고 작성
/troubleshoot  → 문제 발생 시 기록 및 해결
```

### 자동 실행 트리거

| 상황 | 실행할 커맨드 |
|------|--------------|
| 새 유닛 시작 | `/unit-start` |
| 작업 재개 | `/unit-status` → `/unit-start` |
| 테스트 실패 | `/troubleshoot` |
| 테스트 통과 | `/unit-done` |
| 빌드/설치 필요 | `/build` |

---

## 📋 5단계 프로세스 (각 유닛마다 필수)

| 순서 | 단계 | 내용 | 필수 |
|------|------|------|------|
| 1️⃣ | 목표 | 달성할 결과물 정의 | ✅ |
| 2️⃣ | 정의 | 상세 명세 작성 | ✅ |
| 3️⃣ | 개발 | 코드 작성 | ✅ |
| 4️⃣ | 테스트 | 검증 및 통과 확인 | ✅ |
| 5️⃣ | 회고 | 배운 점 기록 | ✅ |

---

## 🚨 금지 사항 (NEVER DO)

- ❌ 슬래시 커맨드 없이 유닛 시작
- ❌ 테스트 없이 다음 유닛으로 진행
- ❌ 문제 해결 후 `/troubleshoot` 기록 누락
- ❌ 회고(`/unit-done`) 단계 생략
- ❌ 여러 유닛 동시 진행

---

## 🔄 표준 개발 흐름

```
[작업 시작]
    │
    ▼
/unit-status  ← 현재 상태 확인
    │
    ▼
/unit-start   ← 5단계 프로세스 시작
    │
    ├─→ 1️⃣ 목표 선언
    ├─→ 2️⃣ 정의 작성
    ├─→ 3️⃣ 코드 개발
    ├─→ 4️⃣ 테스트 실행
    │       │
    │       ├─ PASS → /unit-done → 다음 유닛
    │       │
    │       └─ FAIL → /troubleshoot → 수정 → 재테스트
    │
    └─→ 5️⃣ 회고 작성
```
